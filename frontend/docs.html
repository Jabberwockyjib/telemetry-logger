‚öôÔ∏è Setup Guide

1. Software Installation

Install Python Dependencies

pip install -r requirements.txt

This installs all required Python packages including FastAPI, SQLAlchemy, and hardware communication libraries.

Initialize Database

alembic upgrade head

Creates the database schema and initializes the telemetry tables.

Start the Application

python -m uvicorn backend.app.main:app --host 0.0.0.0 --port 8000 --reload

Starts the Cartelem server with auto-reload for development.

2. Hardware Connection

Connect OBD-II Adapter

  1. Plug the OBD-II adapter into your vehicle's OBD port (usually under the dashboard)
  2. Connect the adapter to your computer via USB
  3. Note the serial port path (e.g., /dev/ttyUSB0 on Linux, COM3 on Windows)

Connect GPS Device

  1. Connect your GPS device to a USB port
  2. Ensure the device is configured for NMEA output
  3. Note the serial port path and baud rate (typically 4800 or 9600)

Connect Meshtastic Device (Optional)

  1. Connect your Meshtastic device via USB
  2. Configure the device for telemetry mode
  3. Set up the radio parameters (frequency, power, etc.)

3. Configuration

Create Environment File

# Database DATABASE_URL=sqlite+aiosqlite:///./cartelem.db # GPS Configuration GPS_PORT=/dev/ttyUSB0 GPS_BAUDRATE=4800 GPS_RATE_HZ=10 # OBD Configuration OBD_PORT=/dev/ttyUSB1 OBD_BAUDRATE=38400 OBD_RATE_HZ=5 # Meshtastic Configuration MESHTASTIC_DEVICE_PATH=/dev/ttyUSB2 MESHTASTIC_RATE_HZ=1 # Application Settings DEBUG=true LOG_LEVEL=INFO

Save this as .env in your project directory and adjust the port paths to match your hardware.

‚ö†Ô∏è Important: Make sure to update the port paths in your configuration to match the actual devices connected to your system. Use ls /dev/tty* on Linux or Device Manager on Windows to find the correct ports.

üîß Configuration Options

Environment Variables

Database Configuration

# SQLite (Development) DATABASE_URL=sqlite+aiosqlite:///./cartelem.db # PostgreSQL (Production) DATABASE_URL=postgresql+asyncpg://user:password@localhost:5432/cartelem

GPS Service Configuration

GPS_PORT=/dev/ttyUSB0 # Serial port path GPS_BAUDRATE=4800 # Baud rate (4800, 9600, 38400) GPS_RATE_HZ=10 # Data collection rate GPS_TIMEOUT=1.0 # Serial timeout in seconds GPS_MAX_RECONNECT=10 # Max reconnection attempts GPS_RECONNECT_DELAY=5.0 # Delay between reconnects

OBD Service Configuration

OBD_PORT=/dev/ttyUSB1 # Serial port path OBD_BAUDRATE=38400 # Baud rate (typically 38400) OBD_TIMEOUT=5.0 # Command timeout OBD_MAX_RECONNECT=5 # Max reconnection attempts OBD_RECONNECT_DELAY=10.0 # Delay between reconnects

Meshtastic Service Configuration

MESHTASTIC_DEVICE_PATH=/dev/ttyUSB2 # Device path MESHTASTIC_BAUDRATE=38400 # Baud rate MESHTASTIC_RATE_HZ=1 # Transmission rate MESHTASTIC_MAX_PAYLOAD=64 # Max payload size

Application Configuration

DEBUG=true # Enable debug mode HOST=0.0.0.0 # Server host PORT=8000 # Server port LOG_LEVEL=INFO # Logging level

OBD PID Configuration

üí° Tip: You can customize which OBD parameters to collect by modifying the PID configuration in the OBD service. Default PIDs include speed, RPM, throttle position, coolant temperature, and engine load.
# Default OBD PIDs DEFAULT_PIDS = { "SPEED": {"rate_hz": 10.0, "unit": "kph", "description": "Vehicle speed"}, "RPM": {"rate_hz": 10.0, "unit": "rpm", "description": "Engine RPM"}, "THROTTLE_POS": {"rate_hz": 5.0, "unit": "%", "description": "Throttle position"}, "COOLANT_TEMP": {"rate_hz": 2.0, "unit": "C", "description": "Coolant temperature"}, "ENGINE_LOAD": {"rate_hz": 5.0, "unit": "%", "description": "Engine load"}, }

üìñ Usage Guide

1. Accessing the Dashboard

Open the Web Interface

Navigate to http://localhost:8000 in your web browser to access the Cartelem dashboard.

You'll see two main interfaces:

  • Live Dashboard - Real-time telemetry visualization
  • Data Replay - Historical session playback

2. Creating a Session

Start a New Telemetry Session

  1. Click "Create New Session" on the dashboard
  2. Enter session details:
    • Session name (e.g., "Track Day 1")
    • Car ID (e.g., "CAR001")
    • Driver name
    • Track/location
  3. Click "Create Session"

3. Starting Data Collection

Begin Telemetry Collection

  1. Select your session from the dropdown
  2. Click "Start Session" to begin data collection
  3. Verify that data is flowing:
    • GPS coordinates updating on the map
    • Charts showing real-time data
    • Status indicators showing "Connected"

4. Monitoring Real-time Data

Live Dashboard Features

  • GPS Map - Real-time position tracking
  • Speed/RPM Chart - Engine performance data
  • Engine Parameters - Temperature, load, throttle
  • Status Cards - Connection status for each service

5. Stopping and Exporting Data

End Session and Export Data

  1. Click "Stop Session" when data collection is complete
  2. Navigate to the "Data Replay" page
  3. Select your session from the dropdown
  4. Use the time scrubber to review the session
  5. Click "Export CSV" or "Export Parquet" to download data

6. Data Replay and Analysis

Historical Data Analysis

  • Time Navigation - Use the scrubber to jump to specific times
  • Playback Controls - Play/pause for smooth data review
  • Chart Analysis - Review performance trends
  • GPS Track - Visualize the complete route
  • Export Options - Download data for external analysis

üîß Troubleshooting

Common Issues and Solutions

OBD-II Adapter Not Detected

Symptoms: OBD service shows "Not Connected" status

Solutions:

  • Check USB connection and try a different port
  • Verify the adapter is compatible with your vehicle
  • Update the port path in configuration (e.g., /dev/ttyUSB0)
  • Check if the adapter requires drivers
  • Test with a different OBD-II adapter
GPS Device Not Responding

Symptoms: GPS service shows "Not Connected" or no position data

Solutions:

  • Ensure GPS device has clear sky view
  • Check baud rate configuration (typically 4800 or 9600)
  • Verify NMEA output is enabled on the device
  • Test with a different GPS device
  • Check serial port permissions
WebSocket Connection Issues

Symptoms: Dashboard shows "Disconnected" status

Solutions:

  • Refresh the web page
  • Check if the server is running
  • Verify firewall settings
  • Try accessing from a different browser
  • Check server logs for errors
Database Connection Errors

Symptoms: Application fails to start or data not saving

Solutions:

  • Check database file permissions
  • Verify database URL configuration
  • Run database migrations: alembic upgrade head
  • Check disk space availability
  • Restart the application
Performance Issues

Symptoms: Slow response times or data lag

Solutions:

  • Reduce data collection rates
  • Check system resources (CPU, memory)
  • Optimize database configuration
  • Use SSD storage for better performance
  • Close unnecessary applications
